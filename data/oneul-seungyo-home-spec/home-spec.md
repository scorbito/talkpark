# 오늘은 승요 - 홈 화면 리디자인 스펙

## 1. 목적
이 화면은 야구 직관 관리 서비스의 메인 홈 화면이다.
단순 정보 모음이 아니라, 사용자의 직관 기록과 다음 행동을 중심으로 보여주는 개인 허브 화면이어야 한다.

핵심 목표:
1. 내 직관 승률을 가장 강하게 보여준다.
2. 직관 등록 / 후기 작성 / 공유를 핵심 행동으로 배치한다.
3. 다음 직관, 최근 직관 경기, 우리팀 일정은 보조 정보로 정리한다.
4. 전체적으로 다크 프리미엄 스포츠 대시보드 느낌을 준다.

---

## 2. 전체 컨셉
- 다크 프리미엄 스포츠 대시보드
- 야간 경기장 감성
- 공식 팀 로고 사용 금지
- 통일된 이니셜 배지 시스템 사용
- 오렌지는 CTA 강조용
- 승/패는 작은 점 + 텍스트 중심
- 전체적으로 고급스럽고 정돈된 느낌
- 정보가 과하게 빽빽해 보이면 안 됨

---

## 3. 화면 구조
홈 화면은 아래 구조를 가진다.

1. Header
2. Hero Card
3. Next Game Card
4. Recent Attendance Games Section
5. Team Weekly Schedule Section
6. Bottom Navigation

팀 순위는 홈 화면에서 제외한다.

---

## 4. 우선순위
이 화면에서 사용자가 가장 먼저 보게 될 순서는 다음과 같다.

1. 내 직관 승률 (.500)
2. 직관 등록 버튼
3. 후기 작성 버튼
4. 다음 직관 정보
5. 최근 직관 경기
6. 우리팀 일정

---

## 5. 섹션별 상세 요구사항

### 5-1. Header
구성:
- 중앙 타이틀: 오늘은 승요

규칙:
- 알림 아이콘 없음
- 좌우 보조 아이콘 없음
- 헤더는 단순해야 하며, 히어로를 방해하면 안 됨

---

### 5-2. Hero Card
목적:
- 홈 화면의 핵심 카드
- 가장 먼저 눈에 들어와야 함

구성:
- 좌상단: 팀 배지 + "내 팀 두산"
- 중앙 상단: "내 직관 승률"
- 중앙 메인 숫자: ".500"
- 중앙 하단: "총 6경기 · 3승 3패 0무"
- 보조 칩: "직관 6", "후기 4"
- CTA 3개:
  - 1순위: "+ 직관 등록"
  - 2순위: "후기 작성"
  - 3순위: "공유"

규칙:
- 내부 요소는 위아래로 너무 붙지 않게 한다.
- .500은 가장 강한 계층으로 보여준다.
- "내 직관 승률"은 보조 헤딩 수준으로 약하게 보여준다.
- "총 6경기 · 3승 3패 0무"는 보조 텍스트로 정리한다.
- "3승"은 녹색, "3패"는 빨강, 나머지는 보조 흰색/회색.
- 직관 / 후기 칩은 버튼처럼 보이지 않게, 작은 정보 칩처럼 처리한다.
- CTA 버튼은 같은 줄에서 높이를 통일하되 우선순위가 보여야 한다.
- Hero 배경은 어두운 야간 경기장 이미지 + 어두운 오버레이 조합 가능.
- 중앙 텍스트 뒤는 너무 복잡하지 않아야 한다.

---

### 5-3. Next Game Card
목적:
- 실용적인 정보 카드
- 빠르게 읽혀야 함

구성:
- 섹션 타이틀: 다음 직관
- 상태 배지: D-1 또는 D-Day
- 날짜 / 시간
- 장소
- 양 팀 매치업
  - 좌측 팀: 배지 + 팀명 + 홈/원정 칩
  - 중앙: VS
  - 우측 팀: 배지 + 팀명 + 홈/원정 칩

규칙:
- 정보 구조는 아래 순서를 지킨다.
  1. 제목 + 상태 배지
  2. 날짜 / 시간 + 장소
  3. 매치업
- VS는 팀명보다 약하게 보여준다.
- 홈/원정 칩은 작고 얇게 처리한다.
- 카드 우측 상단 또는 우측 영역에 상세 이동 chevron 허용 가능.

---

### 5-4. Recent Attendance Games
목적:
- 최근 직관 경기 흐름을 간단히 보여주는 정보 섹션

구성:
- 섹션 제목: 최근 직관 경기
- 보조 요약: 예) 3승 2패
- 우측 링크: 더보기
- 경기 카드 목록

각 경기 카드 포함 정보:
- 날짜
- 양 팀 배지
- 점수
- 결과(승/패) - 작은 점 + 텍스트

표현 규칙:
- 점수 중심 구조 예: D 6 : 3 K
- 긴 팀명 전체 표시는 지양
- 줄임표가 생기지 않게 한다
- 승/패는 아래쪽에 작은 상태 표시로 둔다
- 한 번에 4개가 보여도 너무 촘촘하지 않게 한다
- 필요하면 가로 스크롤 구조로 바꿔도 된다

---

### 5-5. Team Weekly Schedule
목적:
- 이번 주 우리팀 일정 흐름을 간단히 보여주는 보조 섹션

구성:
- 섹션 제목: 우리팀 일정
- 우측 링크: 전체 일정
- 주간 일정 셀 목록
- 하단 요약 문구

각 일정 셀에 들어갈 정보:
- 요일
- 날짜
- 상대팀 배지
- 상태 1개 (오늘 / 예정 / 승 / 패)

규칙:
- 한 셀 안에 정보가 너무 많으면 안 된다
- 홈/원정/승/패/예정을 모두 강하게 같이 넣지 않는다
- 상태는 1개만 핵심적으로 보이게 한다
- "오늘" 셀은 오렌지 outline으로 강조한다
- 하단 요약 예: 5경기 원정 · 1경기 홈
- 하단 요약은 muted 스타일로 처리한다

---

### 5-6. Bottom Navigation
구성:
- 홈
- 일정
- 커뮤니티
- 마이

규칙:
- 현재 활성 탭은 홈
- 활성 색상은 오렌지
- 비활성은 회색/반투명 흰색
- 다크 배경과 자연스럽게 어울리게 한다

---

## 6. 공통 규칙

### 6-1. 배지 시스템
- 공식 팀 로고 사용 금지
- 원형 배지 사용
- 동일한 크기 체계 사용
- 팀 구분은 이니셜 1글자 사용
- 팀별 색은 배지 내부 포인트나 얇은 링으로만 표현

예:
- 두산 D
- LG L
- 삼성 S
- 키움 K
- 한화 H
- NC N
- SSG S
- KT K
- 롯데 L
- KIA K

---

### 6-2. 버튼 규칙
- primary: 오렌지 채움
- secondary: 다크 솔리드
- tertiary: 아웃라인 또는 매우 약한 다크 버튼

우선순위:
1. 직관 등록
2. 후기 작성
3. 공유

---

### 6-3. 텍스트 계층
가장 강한 것:
- .500
- 직관 등록
- 섹션 제목

중간:
- 팀명
- 날짜/시간
- 다음 직관 제목

약한 것:
- 장소
- 홈/원정
- 더보기
- 전체 일정
- 보조 설명

---

## 7. 반드시 피해야 할 것
- 공식 구단 로고 사용
- 전체 UI를 팀 컬러로 과하게 칠하기
- 카드 안에 정보를 너무 빽빽하게 넣기
- 홈 화면에서 순위 테이블 노출
- 보조 정보가 메인 정보보다 강하게 보이는 것
- 최근 경기 카드에서 줄임표가 생기는 레이아웃

---

## 8. 최종 목표
수정 후 화면은 아래 인상을 줘야 한다.

- 프리미엄한 다크 스포츠 앱
- 내 직관 기록이 중심인 홈 화면
- 고급스럽고 정돈된 느낌
- 여백이 살아 있는 UI
- 구현 가능한 모바일 웹 UI
