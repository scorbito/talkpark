# 프로필 모달과 자기소개 기획

> 커뮤니티에서 작성자의 프로필 사진이나 닉네임을 눌렀을 때 표시하는 가벼운 정보 모달 기획입니다.
> 시즌 레벨, 자기소개, 친구 신청 동선을 한곳에서 확인하되, 목록/본문 화면의 스캔성을 해치지 않는 것이 목적입니다.

관련 문서:
- 시즌 레벨 기획: [season-level.md](./season-level.md)
- 기능 명세: [feature-overview.md](./feature-overview.md)
- 작업 인덱스: [../WORKPLAN.md](../WORKPLAN.md)

## 1. 목적

타인의 레벨/직관 활동을 보고 싶을 때 진입하는 가벼운 정보 모달입니다. 커뮤니티 스캔 흐름을 깨끗하게 두고, 호기심이 있는 사용자만 능동적으로 확인하도록 합니다.

## 2. 진입 트리거

- 작성자 닉네임 탭
- 작성자 프로필 사진 탭
- 두 동작 모두 같은 모달을 연다.
- 적용 화면: 후기 상세, 후기 카드, 댓글, 경기톡 글/댓글 등 작성자가 표시되는 모든 영역.
- 본인 프로필을 눌러도 똑같이 모달이 열린다. 특수 분기는 만들지 않는다.

## 3. 표시 항목

| 영역 | 항목 |
|---|---|
| 헤더 | 프로필 사진, 닉네임, 응원팀 뱃지 |
| 레벨 | 현재 시즌 레벨 + 칭호, 예: `Lv.6 응원단골` |
| 자기소개 | `profiles.bio` 텍스트 |
| 활동 지표 | 현재 시즌 직관 횟수, 시즌 직관 승률, 후기 작성 수 |
| 액션 | 친구 신청 버튼 |

## 4. 모달 UX

- 작은 카드형 모달로 표시한다. 전체 화면 모달은 사용하지 않는다.
- 배경 탭 또는 닫기 버튼으로 닫힌다.
- 친구 관계 상태는 기존 `lib/actions/friends.ts`의 `relationship` 값과 1:1 매칭한다.

친구 관계 표시:

- `self`: 본인. 친구 버튼 자리에 "내 프로필" 라벨 또는 버튼 숨김. 대신 "프로필 편집" 진입점 제공도 검토.
- `friend`: "친구" 비활성 뱃지. 길게 누르거나 더보기에서 "친구 끊기" 제공 검토.
- `requested`: 내가 보낸 신청. "신청됨" 비활성 표시. 취소 옵션은 MVP 이후.
- `incoming`: 상대가 나에게 보낸 신청. "수락" / "거절" 버튼 제공.
- `none`: "친구 신청" 활성 버튼 제공.

모달은 새 친구 시스템을 만들지 않고 기존 액션을 재사용합니다.

## 5. 자기소개 필드

- 컬럼: `profiles.bio text NULL`
- 최대 글자 수: 150자
- 줄바꿈: 허용하지 않는다. 한 줄 입력으로 제한한다.
- 이모지: 허용한다.
- 빈 값: 허용한다.
- 빈 슬롯 표시: "아직 소개가 없어요"

마이페이지의 프로필 편집 UI에 자기소개 입력 필드를 추가합니다.

## 6. 데이터 조회

`getPublicProfile(userId)` 서버 액션을 신설합니다.

반환 형태 초안:

```text
{
  user_id,
  nickname,
  avatar_url,
  main_team_id,
  bio,
  season_level: { level, title, total_xp },
  season_stats: { attended, win_rate, review_count },
  relationship: 'self' | 'none' | 'friend' | 'requested' | 'incoming',
  incoming_request_id: string | null
}
```

메모:

- `relationship`은 기존 `FriendCandidate.relationship` 값과 동일 enum에 `self`를 추가한다.
- `incoming_request_id`는 `respondFriendRequestAction(requestId, ...)` 호출에 필요하다.
- 캐싱은 도입하지 않고, 모달을 열 때마다 가볍게 조회한다.

## 7. 개발 우선순위

본 모달과 자기소개는 시즌 레벨 본 기능보다 먼저 도입합니다.

이유:

- 후기 상세에 레벨을 상시 노출하지 않는 대신 닉네임/사진 탭 → 모달 인터랙션을 먼저 깔아둔다.
- 시즌 레벨이 붙을 자리를 미리 준비할 수 있다.
- 자기소개 자체는 시즌 레벨과 무관하게 독립 가치가 있다.

권장 순서:

1. `profiles.bio` 컬럼 추가 + 프로필 편집 UI 자기소개 입력.
2. `getPublicProfile` 서버 액션 + 프로필 모달 컴포넌트.
3. 후기/댓글/경기톡 작성자 영역 탭 → 모달 진입 연결.
4. 시즌 레벨 본 기능 개발 시 모달의 레벨 영역을 실데이터로 연결.
