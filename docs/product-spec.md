# 오늘은 승요 Product Spec

이 문서는 다른 작업자나 에이전트가 `docs/WORKPLAN.md`의 작업 인덱스를 읽기 전에 제품의 목적, 기능, 정책, 현재 구현 범위를 빠르게 파악하기 위한 요약 문서다. 작업 히스토리와 진행 로그는 `docs/WORKPLAN.md`와 `docs/planning/*`를 기준으로 보고, 이 문서는 제품/기능 기준의 현재 스펙으로 유지한다.

## 1. 제품 요약

오늘은 승요는 KBO 직관 팬을 위한 모바일 우선 웹앱이다.

핵심 경험은 다음 네 가지다.

- 내가 간 경기와 응원팀을 기록한다.
- 티켓 사진으로 직관을 인증하고 승/패/무 결과를 쌓는다.
- 내 직관 승률과 티켓 컬렉션을 확인한다.
- 후기를 커뮤니티에 공유하고 친구와 소통한다.

## 2. 핵심 사용자

- KBO 직관 기록을 따로 모아보고 싶은 팬
- "내가 가면 이긴다"는 승요 경험을 수치로 보고 싶은 팬
- 티켓 사진, 경기 결과, 후기를 한 곳에 남기고 싶은 팬
- 친구나 다른 팬의 직관 후기를 보고 반응하고 싶은 팬

## 3. 제품 원칙

- 모바일 앱처럼 느껴지는 웹앱을 우선한다.
- KBO 구단 로고와 상표 이미지는 사용하지 않고, 팀명/팀 컬러/이니셜 배지로 표현한다.
- UI 텍스트와 핵심 컴포넌트는 HTML/CSS로 구현한다.
- GPT 생성 이미지는 배경, 마스코트, 후기 이미지, 공유 카드 배경 용도로만 사용한다.
- 사용자는 가입 전에도 익명 세션으로 먼저 써볼 수 있어야 한다.
- 실제 서비스 데이터는 Supabase Auth/DB/Storage를 기준으로 관리한다.

## 4. 주요 기능

### 인증과 시작

- 랜딩에서 정식 로그인 또는 익명 시작을 선택한다.
- 이메일/비밀번호, Google OAuth, 카카오 OAuth를 지원한다.
- 익명 사용자는 기본 프로필이 자동 생성된다.
- 익명 사용자는 이후 이메일 또는 OAuth로 정식 계정 전환이 가능하도록 설계되어 있다.
- 신규 사용자는 온보딩에서 닉네임, 내 팀, 관심팀을 설정한다.

### 홈

- 내 직관 승률과 승/패/무 통계를 보여준다.
- 다음 직관 또는 현재 진행 중인 직관을 강조한다.
- 최근 직관 기록을 카드로 보여준다.
- 우리 팀의 이번 주 일정을 요약한다.
- 새 공지가 있으면 헤더 알림 액션에 unread 표시를 보여준다.

### 직관 등록과 인증

- 예정 직관과 이전 직관을 등록할 수 있다.
- 날짜, 경기, 응원팀, 메모, 티켓 사진을 입력한다.
- 티켓 사진은 Supabase Storage에 업로드한다.
- Gemini Vision으로 티켓의 경기일자, 홈팀, 원정팀, 구장을 OCR/구조화한다.
- Vision 결과와 DB의 KBO 경기 데이터를 매칭해 직관 인증을 처리한다.
- 동일 티켓 이미지 hash와 동일 사용자+동일 경기 중복 등록을 차단한다.
- 미인증 직관에는 나중에 티켓을 추가해 인증할 수 있다.

### 경기 종료와 결과

- 직관 경기 시간이 지난 뒤 사용자가 결과 확인을 요청할 수 있다.
- DB에 종료 결과가 있으면 즉시 반영한다.
- 결과가 없으면 KBO 동기화 route를 호출해 해당 날짜 경기를 갱신한다.
- 결과는 승/패/무에 따라 축하/위로/무승부 모달로 보여준다.
- 결과 모달에서 바로 후기 작성으로 이어질 수 있다.

### 일정

- KBO 시즌 일정을 월간 캘린더로 탐색한다.
- 기본 보기와 시리즈 보기를 제공한다.
- 시리즈 보기는 주별 grid와 팀별 막대, 시리즈 결과 행을 사용한다.
- 일정 데이터는 KBO 공식 API를 우선 사용하고, 네이버 스포츠를 fallback 소스로 둔다.
- `games` 테이블은 외부 경기 ID 기준으로 idempotent upsert한다.

### 팀 순위

- `team_standings` 테이블의 시즌 순위를 읽어 보여준다.
- 승률, 경기 차, 승-무-패, 최근 5경기 form을 표시한다.
- 순위 동기화는 KBO/네이버 데이터를 기반으로 한다.

### 커뮤니티와 후기

- 사용자는 직관 1건당 후기 1개를 작성할 수 있다.
- 후기에는 본문, 사진 1~3장, 공개 범위를 저장한다.
- 공개 범위는 전체 공개, 친구 공개, 나만 보기다.
- 익명 사용자는 전체 공개만 선택할 수 있다.
- 후기는 커뮤니티 피드, 내 후기 모음, 후기 상세에서 조회한다.
- 좋아요, 저장, 댓글, 공유를 지원한다.
- 댓글은 flat 리스트이며, 본인 댓글 또는 후기 작성자가 삭제할 수 있다.
- 후기 본문에서 hashtag를 자동 추출하지만, 현재 UI에서는 칩 노출을 제거했다.

### 공유 카드

- 내 직관/후기 결과를 9:16 공유 카드로 미리보기한다.
- 템플릿 3종을 제공한다.
- `html2canvas`로 카드 DOM을 PNG로 캡처한다.
- 지원 브라우저에서는 Web Share API로 이미지 파일을 공유한다.
- 미지원 환경에서는 PNG 다운로드와 텍스트 클립보드 복사를 fallback으로 제공한다.

### 마이 페이지

- 프로필, 내 팀, 관심팀, 직관 통계, 메뉴를 보여준다.
- 프로필 사진과 닉네임, 팀 설정을 편집할 수 있다.
- 내 직관 리스트, 내 후기 모음, 내 티켓 컬렉션, 친구 관리, 설정으로 이동한다.
- 익명 사용자에게는 정식 계정 전환 CTA를 보여준다.

### 티켓 컬렉션

- 인증된 직관의 티켓 이미지를 모아 보여준다.
- `ticket-images` 버킷이 private이므로 signed URL을 발급해 표시한다.
- 티켓 이미지는 그리드와 전체 화면 줌 모달로 확인한다.

### 친구

- 친구는 follow가 아니라 양방향 friend 관계다.
- 친구 신청, 수락, 거절 흐름을 둔다.
- 익명 사용자는 친구 기능을 사용할 수 없고 정식 계정 전환 CTA를 본다.
- 친구 공개 후기는 작성자와 친구 관계인 사용자에게만 노출한다.

### 운영 페이지

- 공지 목록과 공지 상세를 제공한다.
- 공지는 Supabase `notices` 테이블 기반이다.
- 이용안내와 FAQ를 제공한다.
- 문의하기는 MVP에서 mailto와 Gmail compose fallback을 사용한다.
- 이용약관과 개인정보처리방침 페이지를 제공한다.

## 5. 데이터와 정책

### 주요 모델

- `profiles`: 사용자 프로필, 내 팀, 관심팀, 알림, 기본 공개 범위, 프로필 이미지
- `teams`: KBO 팀 기본 정보와 색상/이니셜
- `games`: KBO 경기 일정, 점수, 상태, 경기장
- `team_standings`: 시즌 순위와 최근 form
- `attendances`: 사용자의 직관 기록과 티켓 인증 정보
- `reviews`: 직관에 종속된 후기
- `review_likes`, `review_saves`, `review_comments`: 후기 반응과 댓글
- `friends`, `friend_requests`: 양방향 친구와 신청 흐름
- `notices`: 운영 공지

### 핵심 제약

- 한 사용자는 같은 경기 직관을 한 번만 등록할 수 있다.
- 한 직관에는 후기 하나만 작성할 수 있다.
- 티켓 이미지 hash는 중복 인증을 막기 위해 unique 처리한다.
- 관심팀은 최대 5개다.
- 익명 사용자는 친구 기능과 friends/private 후기 공개가 제한된다.

### 승률 계산

- 승률은 사용자가 응원한 팀 기준으로 계산한다.
- 무승부는 승률 분모에서 제외한다.
- 경기 결과가 끝난 직관만 승/패/무 결과에 반영한다.

- `/landing`: 랜딩, 정식 로그인/익명 시작 진입
- `/login`: 이메일/Google/카카오 로그인, 익명 계정 업그레이드 진입
- `/onboarding`: 닉네임/내 팀/관심팀 설정

- `/rankings`: 팀 순위

- `/reviews/[id]`: 후기 상세와 댓글

- `/my/contact`: 문의하기
- `/legal/terms`: 이용약관
- `/legal/privacy`: 개인정보처리방침

## 7. 외부 연동

- Supabase Auth: 이메일, Google, 카카오, 익명 로그인
- Supabase Database: 서비스 데이터 저장
- Supabase Storage: 티켓, 후기, 프로필 이미지 저장
- Gemini Vision: 티켓 이미지 OCR과 경기 정보 추출
- KBO 공식 API: 일정 동기화 1순위
- 네이버 스포츠: 일정/순위 fallback
- Vercel Cron: KBO 일정/순위 갱신 route 호출

## 8. 운영 단계 후속 항목

- MVP 출시와 사용자 모집 단계 진입 기준으로 핵심 기능 검토는 완료됐다.
- Vercel Cron, KBO/네이버 응답 변화, Supabase/Vercel 에러 로그는 운영 중 관측한다.
- 접근성, 색대비, E2E 자동화는 출시 차단 항목이 아니라 운영 품질 개선 항목으로 관리한다.
- 이용약관과 개인정보처리방침은 게시되어 있으며, 법무 검토와 회원가입 동의 체크박스는 운영 중 보강한다.
- 경기 상세 페이지는 현재 보류 상태이며 `/schedule`로 redirect한다.
- 초기 Supabase 적용 문서는 삭제했으며, 현재 DB/운영 기준은 `supabase/*.sql`, `.env.example`, Vercel 환경변수, 실제 server action 코드를 기준으로 확인한다.
