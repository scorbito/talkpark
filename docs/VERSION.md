## 1. 현재 버전

- 현재 버전: `v0.11.0`
- 상태: Public MVP / Beta
- 기준일: 2026-05-16
- 단계: 공개 MVP 이후 두 번째 주요 기능 추가 릴리즈

`v0.11.0`은 시즌 레벨 시스템과 프로필 모달, 한 줄 자기소개를 추가한 베타 기능 확장 버전입니다. 직관 기록과 후기 작성이 시즌 단위 성장감으로 이어지도록 했고, 커뮤니티 곳곳의 작성자 영역에서 다른 사용자의 응원팀과 자기소개, 시즌 활동을 가볍게 확인하고 친구 신청까지 한 자리에서 처리할 수 있게 했습니다.

아직 정식 `v1.0.0`으로 보지 않는 이유는 운영 관측성, 법무/동의 보강, 운영 문서화, 첫 사용자 피드백 반영이 남아 있기 때문입니다.

## 2. 버전 정책

프로젝트는 SemVer를 가볍게 따르되, 제품 단계에 맞춰 아래 기준으로 운영합니다.

- `v0.x`: MVP, 베타, 사용자 피드백 반영 단계
- `v0.9.x`: 공개 MVP 이후 버그 수정, 운영 안정화, 작은 UX 개선
- `v0.10.x`: 공개 MVP 이후 의미 있는 기능 추가 또는 주요 UX 개선
- `v1.0.0`: 정식 출시 기준 충족

변경 단위:
- patch: 버그 수정, 문구 수정, 작은 UI/UX 개선, 운영 설정 보강
- minor: 새 기능 추가, 주요 사용자 흐름 개선, 운영 기능 추가
- major: 큰 제품 방향 변경, 데이터 구조나 호환성에 영향을 주는 변경

## 3. v1.0.0 기준

아래 항목이 충분히 정리되면 `v1.0.0` 정식 출시로 올릴 수 있습니다.

- 실제 사용자 피드백 1차 반영
- Vercel/Supabase/KBO 동기화 운영 로그 안정 확인
- 이용약관/개인정보처리방침 법무 검토
- 회원가입 또는 계정 전환 시 동의 체크박스 도입
- 환경변수와 seed/SQL 적용 순서 문서화
- README 실행/배포/운영 기준 보강
- 치명적인 핵심 흐름 이슈 없음

## 4. 릴리즈 기록

### v0.11.0 - 2026-05-16

시즌 레벨 시스템과 프로필 모달, 한 줄 자기소개를 함께 추가한 릴리즈.

추가 기능:
- 시즌 레벨 시스템 도입 (Lv.1 새싹직관러 ~ Lv.10 레전드승요, 총 10단계)
- 시즌 기준은 `games.game_date`의 연도, 매년 새 시즌으로 리셋
- XP 지급: 경기 결과 확인 +30, 티켓 인증 +100, 후기 작성 +70, 후기에 직접 찍은 사진 첨부 보너스 +20
- XP 회수: 후기 삭제, 직관 삭제, 티켓 인증 취소 등 반대 행동 시 멱등하게 회수
- 이벤트 로그 기반의 `season_xp_events` 테이블로 XP 합계 관리 (단순 합계 컬럼 X)
- 1일 1직관 trigger 도입: `attendances` insert/update 시 같은 사용자의 같은 경기일 직관 차단
- 기존 사용자 활동에 대한 시즌 XP 백필 스크립트 실행
- 마이페이지 프로필 영역에 시즌 레벨 카드 추가 (현재 레벨/칭호, 현재 시즌 XP, 다음 레벨까지 필요한 XP, 진행 바)
- 홈 내 카드에 짧은 레벨 표시 추가
- 마이페이지 프로필 정리: 연동 계정 정보 → 설정 페이지로 이동, 프로필 편집은 작은 아이콘 버튼으로 축소
- 한 줄 자기소개(`profiles.bio`) 도입: 최대 150자, 한 줄, 이모지 허용
- 마이페이지 프로필 편집 UI에 자기소개 입력 추가
- 프로필 모달 추가: 후기 상세 / 후기 카드 / 댓글 / 경기톡 글 / 경기톡 댓글의 작성자 닉네임·사진 탭 시 진입
- 프로필 모달에 응원팀 뱃지, 자기소개(빈 값은 "아직 소개가 없어요"), 시즌 직관 활동(횟수·승률·후기 수), 시즌 레벨/칭호 표시
- 프로필 모달 친구 액션: 본인/none/friend/requested/incoming 5가지 상태 처리, 기존 `friends.ts` 액션 재사용
- 본인 프로필을 눌러도 동일하게 모달 표시 (특수 분기 없음)
- 후기 상세·커뮤니티·댓글·경기톡 작성자 영역에는 레벨을 상시 노출하지 않음 (모달 진입 시에만 확인)
- `getPublicProfileAction` 서버 액션 신설 — 닉네임/사진/팀/bio/시즌 레벨/시즌 활동/관계 상태 한 번에 조회

운영 변경:
- 시즌 레벨용 SQL 마이그레이션 적용 (`supabase/season-level.sql`, `add-attendance-date-trigger.sql`, `add-profile-bio.sql` 등)
- `season_xp_events` RLS: 본인 row SELECT만, INSERT/UPDATE/DELETE는 service role만
- 기존 공지 2건의 핀 고정 해제 (시즌 레벨 출시 공지 발행 시점에 정렬 정리)

공지사항 요약:
- "직관할수록 시즌 레벨이 올라갑니다. 티켓 인증을 하면 더 빠르게 성장합니다."
- "닉네임이나 프로필 사진을 누르면 상대방의 응원팀, 자기소개, 직관 활동을 확인하고 친구 신청도 한 자리에서 보낼 수 있습니다."
- "마이 → 프로필 편집에서 한 줄 자기소개(150자)를 추가할 수 있습니다."

운영 확인:
- 시즌 레벨 기획 / 개발 계획 문서 검토 완료
- 시즌 레벨 Step 0~11 완료 기록 확인 (Step 0 프로필 모달 + 자기소개 선행 출시)
- `feature/season-level` master 병합 및 운영 배포 완료

### v0.10.0 - 2026-05-14

경기톡 기능 추가 릴리즈.

추가 기능:
- 커뮤니티에 [후기] / [경기톡] 서브탭 추가
- 특정 KBO 경기에 묶이는 경기톡 글 작성, 조회, 삭제
- 로그인 사용자의 경기톡 작성 지원
- 비로그인 사용자의 경기톡 목록 조회 허용
- 본문 300자 제한, 감정 태그 1개 선택
- 사진 0~1장 첨부 지원
- 작성 시점의 경기 점수와 상태를 글에 저장
- 경기톡 좋아요와 댓글 지원
- 후기 댓글 UI를 공용 `CommentThread` 흐름으로 정리해 경기톡에서도 재사용
- 경기톡 카드에 응원팀, 직관 인증, 점수, 경기 상태, 좋아요 수, 댓글 수 표시
- 홈의 이번 주 경기 카드에서 경기톡 글 수 배지 표시
- 일정 화면에서 해당 경기의 경기톡으로 이동하는 링크 추가
- 경기톡 전용 DB 테이블과 RLS 정책 추가
- 라이브 스코어 lazy refresh 기준 추가

공지사항 요약:
- "경기별로 모여 이야기하는 경기톡이 추가되었습니다."
- "경기 중 응원, 분노, 불안, 환호 같은 감정을 짧게 남길 수 있습니다."
- "홈과 일정에서 바로 해당 경기의 경기톡으로 이동할 수 있습니다."

운영 확인:
- 경기톡 기획 문서 검토 완료
- 경기톡 개발 계획 기준 Step 0~9 완료 기록 확인
- `feature/match-talk` master 병합 및 운영 배포 완료 기록 확인
